# Domain Docs

This repository uses a multi-context domain layout.

## Before exploring

- Read `CONTEXT-MAP.md` at the repository root.
- Read the `CONTEXT.md` files relevant to the modules being changed.
- Read system-wide decisions in `docs/adr/` and module-specific ADRs when present.
- Use the domain terms defined by those files in code, tests, plans, and reports.
- Explicitly flag any proposal that conflicts with an ADR.

If a referenced domain file does not exist yet, proceed without treating its absence as an error.
