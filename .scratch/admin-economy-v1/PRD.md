# 早期运营、装备随机、抽奖与悬赏任务计划

## 状态

已确认设计；尚未开始实现。

## 目标

为 Aeonblight 的早期上线提供可审计的超级管理员运维能力，并补齐：

- Solana AEB 付费主抽奖；
- 可版本化、实时重算的装备白值/副词条/强化体系；
- 四品级采集材料、Boss 券和 Boss 准入；
- 悬赏任务板与徽章抽奖；
- NPC 装备回收与七天回溯窗口；
- 数据库权威的原子配置发布与回滚。

## 已确认的产品规则

### 管理与回溯

- `/api/admin/ops/**` 复用新项目的 `SUPER_ADMIN` 认证，不引入旧项目的独立密钥。
- 所有写操作必须有稳定 `opId` 和非空 `reason`，幂等、事务化，并写审计。
- 首期覆盖：玩家数据修正、补发、反向调整/回溯、批量补偿、支付订单追踪/补发、服务器/队列/在线玩家运维、管理员代抽。
- 回溯仅在目标资产仍可安全定位且未流转时自动执行；已消耗、提现或交易的资产拒绝自动扣回，改走显式人工补偿/调整，绝不制造负余额或负库存。
- 高风险写操作必须先 `dryRun`，再携带显式确认字段提交；审计记录认证操作者、原因、前后状态和关联业务 ID。

### 配置发布

- JSON 配置是 Bootstrap 默认值；数据库中的激活配置发布包是线上唯一权威。
- 发布包可同时改动物品、装备、词条、掉落、任务、抽奖、Boss 券等配置；必须整体校验、预览影响后原子激活。
- 每一版数据库配置保存不可变内容、差异、校验、操作者、原因、时间和审计；任何历史发布包都可重新激活以回滚。
- 装备及 NFT 仅保存可重算语义；激活配置后，已有装备/NFT 立即按新配置重算。
- 账号服是最终装备数值唯一权威；经济快照、装备详情、交易行和 NFT 均返回同一份已解析属性。

### 装备、坐骑与强化

- 装备槽位：武器、头盔、胸甲、手套、鞋子、披风、饰品、坐骑。
- 武器类型：剑、斧、弓、法杖；旧匕首改为斧头，旧裤子改为鞋子，旧布甲改为手套。
- 普通装备阶段：`1/5/10/15/20/25/30`；品级：白、绿、蓝、紫、金、红。
- 可成长平面白值（攻击、护甲、生命）按：
  - 阶段倍率：`1 / 1.40 / 1.96 / 2.74 / 3.84 / 5.38 / 7.53`；
  - 品级倍率：白 `1.00`、绿 `1.10`、蓝 `1.22`、紫 `1.40`、金 `1.65`、红 `1.95`。
  - 因此本阶段紫色白值等于下一阶段白色白值。
- 百分比白值由每个装备模板逐阶段显式配置，不套上述倍率：1/5 级无暴击和暴伤，10 级开始可有暴击率/闪避，15 级开始可有暴伤；吸血和自动回血永远不属于白值。
- 斧头的 1 级白装定位为高单击爆发，包含 `-8%` 攻速；负面白值允许存在，但副词条只产生正向属性，最终属性计算需要安全下限。
- 副词条：白到红固定 `1/2/3/4/5/6` 条；同 key 最多两条。两条同 key 初始基础值相同，但作为独立实例强化后允许不同。
- 武器副词条池：攻击、攻速、暴击率、暴伤、吸血；非武器池：攻击、防御、生命、攻速、暴击率、暴伤、闪避、自动回血。所有副词条在所有等级均可随机出现，首发权重均为 `1`，后续配置可调。
- 副词条数值不入库。实例保存 `key`、实例标识和被命中的强化次数；当前配置按装备模板/阶段/品级实时解析平面与百分比词条值。
- 强化完整开放：每次必定成功、等概率命中一条副词条实例、不影响白值/品级/词条数量，每次令目标词条在其初始配置值基础上额外增加 `10%`。首发上限 `+10`，配置可提高。
- 强化石：装备使用自身阶段的 `enhancement_stone_t1/t5/t10/t15/t20/t25/t30`。`+1–+5` 仅花金币，`+6–+10` 花金币及对应阶段强化石；品级和当前强化次数决定数量/金币成本。强化石物品与接口本期完成，但暂不配置自然掉落路径。
- 坐骑：五只固定橙色装备，不参与等级、品级、强化或普通副词条。全部固定给予最终攻击 `+5%`、最终生命 `+5%`、移动速度 `+25%`：腐甲尸甲虫、冷牙灰狼、泥沼獠猪、棘鬃战狼、恐角蛮牛。
- 命名系列：缚灰(1)、幽皮(5)、影铁(10)、夜璃(15)、蚀日卫(20)、虚铸(25)、永劫蚀光(30)。

### 材料、Boss 券和回收

- 采集材料族：灰烬木 `ashwood`、幽墓石 `gloomstone`、影铁 `shadow_iron`、暮孢 `gloomcap_spore`；每族白/绿/蓝/紫是四个独立 `itemId`，各掉落权重独立配置。
- Boss 券是紫色、`UNBOUND`、可交易的消耗品。每个 Boss 楼层在现有 `dungeons.json` 的 `enterCost.items` 直接声明专属券；券在采集/副本 `loot_pools.json` 和徽章抽奖中配置产出，不建立独立 Boss 券配置文件。
- NPC 回收：普通装备回收后进入 `NPC_RECYCLED`，七天内仅超级管理员可回溯；到期 Worker 硬删除装备实例，保留不可恢复的审计/流水墓碑。NFT 已请求、已铸造或存在链上关联的装备禁止 NPC 回收。
- NPC 回收金币只按装备模板阶段和品级配置；副词条与强化不加价。

### 主抽奖

- 玩家主抽奖为 Solana AEB `LOTTERY_DRAW` Payment Order，单抽 `30 AEB`，支持 `1–10` 抽。
- 下单时锁定配置发布包 ID、奖池、候选范围、价格和次数；链上付款确认后按快照开奖。奖励进 Loot Tray，付款确认时二次校验容量；不足的未放入奖励以订单 fulfillment 保留并可领取/处理。
- 类别权重：装备 `85%`、稀有材料 `13.5%`、Boss 券 `1%`、坐骑 `0.5%`。
- 装备品级：蓝 `80%`、紫 `15%`、金 `5%`；不产出白、绿、红。装备阶段不高于角色当前等级。
- 稀有材料候选仅含四个材料族的蓝/紫版本，各 item 独立权重可调。

### 悬赏任务板

- 首期只实现旧版悬赏任务板，不实现主线、日常或成就系统。
- 五槽：槽 1 免费；槽 2 为角色永久 `3000 Gold`；槽 3/4/5 为账号级共享的 `300/600/900 AEB` 支付解锁，不要求顺序购买。
- 免费刷新每 4 小时一次；金币刷新 `300 Gold`；高级刷新 `30 AEB`。
- 刷新仅替换未完成、未锁定、未领取任务；高级刷新在存在可替换槽位时保证恰好一项稀有提交装备任务，否则不扣费。
- 免费/金币刷新仅产出普通采集或击杀任务；稀有提交装备任务只由高级刷新产生。
- 采集任务追踪精确 `itemId`（材料族 + 品质），不混合品质。
- 击杀任务由游戏服在副本结束提交击杀事实，账号服按关卡敌人编组/理论上限校验，并按成功 `dungeonRunId` 去重。
- 暂定提交装备任务：消耗一件蓝色及以上、背包中、非 NFT 的普通装备；不要求特定副词条，后续可扩展为指定词条/部位/强化等级。
- 普通任务只奖励 1 个普通悬赏徽章；稀有任务只奖励 1 个稀有悬赏徽章。
- 徽章抽奖即时结算：金币、Boss 券即时发放；AEB 一律写入锁定 AEB。
  - 普通徽章：金币 60%（500–2000）、低阶 Boss 券 30%、锁定 AEB 10%（1–3）；
  - 稀有徽章：金币 45%（1000–5000）、中阶 Boss 券 25%、高阶 Boss 券 15%、锁定 AEB 15%（3–10）。

## 实施顺序

1. 配置发布基础：发布包表、配置校验/激活/回滚、热加载与审计；先覆盖现有 JSON，保持当前路由兼容。
2. 装备域改造：目录、实例语义、解析器、装备/NFT/市场响应、七天 NPC 回收 Worker；补全七阶段十类装备与五只坐骑配置。
3. 副词条与强化：随机生成、实时解析、强化石目录与玩家/管理员接口、NFT 元数据投影；强化石掉落路径明确标为未启用。
4. 四品级材料与 Boss 券：物品目录、掉落池、配方/副本入口、30 层 Boss 准入配置和迁移校验。
5. 主抽奖：订单创建、提交/确认、奖池快照、确定性开奖/交割、订单查询与管理员补发/追踪。
6. 悬赏任务板：槽位支付、刷新、锁定、进度事实、装备提交、徽章与徽章抽奖；扩展副本敌人编组/击杀校验。
7. 超级管理员运维：查询、修正、补发、全服补偿、回溯、支付 replay、服务器/队列/踢人、代抽、配置发布管理。
8. 完整契约、并发、数据库集成和 Worker 回归测试；更新 API 文档、上下文和运行手册。

## 数据与接口设计原则

- 所有经济写路径要有 `opId`，使用数据库唯一约束/幂等回放，不依赖进程内状态。
- 任何随机结果记录随机语义/种子、配置发布包和输出描述，便于审计和支付补发；装备具体数值不入库。
- 管理补发和订单 replay 必须先 dry-run，显示容量、目标、潜在重复交割与不可回溯边界。
- 所有迁移新增 `migrations/updates/`，同时折回 `migrations/aeonblight_full_schema.sql`；应用启动不得执行迁移。
- 每个配置发布包必须验证：物品引用、品级/阶段完整性、权重为正且可归一、掉落/任务/Boss 入口引用、NFT/装备解析一致性。

## 验收重点

- 配置激活/回滚使同一装备在快照、市场和 NFT 中显示同一实时属性。
- 重复副词条初始值相同；强化命中其中一条不会改变另一条。
- 付款未确认不开奖；确认重复回调不重复交割；订单奖池按创建快照而非最新配置。
- 任务进度不能由客户端伪造，副本结算重放/重复击杀不能重复加进度。
- NFT 装备不能回收；普通回收在 7 天内可回溯、到期后只能看到审计墓碑且不可恢复。
- 管理批量操作不是全量 dry-run 成功则不写入；任何一个目标失败都不产生部分补偿。
